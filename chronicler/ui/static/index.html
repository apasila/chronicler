Getting Started

  1. Add a project — click + Add Project and select your project folder
  2. Get an AI key — open Settings ⚙ and paste a Groq API key (free at console.groq.com)
  3. Watch it run — the green dot means Chronicler is watching. Save any file to see your first log entry appear

The Handoff

When you're ready to hand off to an AI coding agent, click ✦ Handoff on any project card. The AI reads your recent session history and generates a briefing document. Paste it at the start of any Claude, Cursor, or Copilot session.

  • Generate after a productive session, not mid-task
  • The more changes logged, the richer the handoff
  • Use it with Cursor rules or Claude's CLAUDE.md for persistent context

Tech Stack Sheet

Chronicler scans your project and generates STACK.md — a snapshot of every library, service, font, colour, and architectural decision. Coding agents read this to avoid introducing conflicting libraries or overriding decisions you've already made.

  • Click ⬡ Stack to regenerate at any time
  • The sheet auto-updates when manifest files change (package.json, pyproject.toml, etc.)
  • Edit .chronicler/tech-stack.json to add constraints — rules that survive every regeneration (e.g. "Do not introduce a second state management library")
  • From the terminal: chronicler stack regenerate

Understanding the Activity Feed

Every file save triggers an AI analysis. Each log entry shows the change type, impact level (red/amber/green dot), file path, and a one-sentence summary. Click any entry to expand the diff.

TypeMeaning
featurenew functionality
bug fixfixing something broken
refactorrestructuring without changing behaviour
configconfiguration and build changes
dependencypackages added, removed, or updated
docsdocumentation changes

CLI Reference

chronicler ui                    # open dashboard
chronicler start                 # start watching in background
chronicler stop                  # stop watching
chronicler handoff               # generate handoff in terminal
chronicler stack regenerate      # regenerate tech stack sheet
chronicler log                   # browse change log
